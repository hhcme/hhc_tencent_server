# HHC Server Manager — 宝塔类产品定位与功能补充

> 本文档用于统一新的产品方向：HHC Server Manager 不再只是云服务器连接工具，而是一个 macOS 原生的宝塔类服务器管理产品。它以 SSH 为核心管理 Linux 服务器，并在用户主动配置云厂商 API 后获得实例发现、云监控、安全组、电源、快照等增强能力。

## 1. 新产品定位

HHC Server Manager 的目标是成为面向个人开发者和小团队的本地原生服务器管理面板。产品体验参考宝塔面板的信息架构和常见运维工作流，但产品形态是 macOS 原生客户端，不是 Web 面板，也不要求服务器安装重量级 agent。

核心原则：

- **SSH-first**：服务器管理能力优先通过 SSH、systemd、远程文件、Nginx、Cron、防火墙等通用 Linux 能力实现。
- **云 API optional enhancement**：腾讯云、阿里云、华为云等云 API 只作为增强层，用于实例发现、云资源元数据、云监控、安全组和电源操作。
- **能力探测优先**：不假设所有服务器都有 Nginx、systemd、MySQL、Docker 或云 API；功能入口必须基于运行时探测启用或降级。
- **本地安全闭环**：服务器配置、SSH 私钥、云凭据和 webhook secret 等敏感信息保存在本地 Keychain 或本地数据库引用中，不默认上传到第三方服务。

本产品不是：

- 不是只管理腾讯云 CVM 的云控制台替代品。
- 不是要求在服务器上长期安装重 agent 的 Web 运维面板。
- 不是通用 CI/CD 平台或 Kubernetes 管理平台。
- 不是完整数据库客户端，数据库深度管理应分阶段实现。

## 2. 目标用户

| 用户类型 | 典型场景 | 产品价值 |
| --- | --- | --- |
| 个人开发者 | 管理一台或多台轻量应用服务器、VPS、自建 Linux 机器 | 在本地客户端完成连接、文件、网站、服务、部署和监控 |
| 小团队运维 | 多人维护网站、API、数据库、私有包仓库 | 统一服务器视图、减少手工命令、保留操作审计 |
| 独立产品开发者 | 部署前后端服务、GitLab、npm 私有仓库、Flutter/Dart 包配置 | 把项目部署和开发服务管理集中在一个工作台 |
| 云服务器用户 | 使用腾讯云轻量应用服务器、CVM、阿里云 ECS、华为云 ECS | SSH 管理为主，云资源信息和安全组等能力作为增强 |

## 3. 服务器类型策略

服务器类型必须作为产品模型的一部分，避免把所有云服务器显示成 CVM。

| 类型 | 展示名称 | 识别来源 | 处理策略 |
| --- | --- | --- | --- |
| 腾讯云轻量应用服务器 | 轻量应用服务器 | 用户手动选择、云导入元数据、实例来源标记 | UI 显示为轻量应用服务器，不显示为 CVM |
| 腾讯云 CVM | CVM | Tencent Cloud CVM API 或用户选择 | 可启用 CVM 云监控、安全组、电源等增强能力 |
| 阿里云 ECS | ECS | Alibaba Cloud ECS API 或用户选择 | 走统一云 provider adapter |
| 华为云 ECS | ECS | Huawei Cloud ECS API 或用户选择 | 走统一云 provider adapter |
| 自建服务器/VPS | 自建服务器 | 手动 SSH 添加 | 仅依赖 SSH 能力探测 |
| 未知 Linux 服务器 | 手动 SSH | 缺少云元数据时的默认类型 | 不阻塞 SSH 管理能力 |

产品要求：

- 工作台标题、服务器详情、服务器切换器和列表都应显示准确的服务器类型。
- 云 provider 不可用时，服务器类型不能回退成错误的 CVM。
- 云 API 只补足云侧能力，不影响 SSH 首页、文件、服务、网站、流量监控等主路径。

## 5. 首页和流量监控展示要求

首页是产品改版后的第一感知界面，应优先做成可扫描、可判断、可操作的服务器状态页。

### 5.1 首页信息结构

- 顶部服务器摘要：服务器名称、类型、IP、系统、运行时长、连接状态、最近刷新时间。
- 资源圆环：CPU、内存、磁盘、负载，使用圆环进度展示占用比例或风险等级。
- 流量卡片：当前上行/下行速率、累计发送/接收、活跃网卡。
- 趋势图：CPU、内存、网络上下行的短期折线图，优先使用本地持久化的 Dashboard 快照。
- 能力标签：`/proc`、systemd、sftp、nginx、firewall、docker、mysql、redis 等探测结果。
- 服务概览：运行中服务、失败服务、常见应用服务数量。
- 风险提示：主机密钥未信任、Nginx 测试失败、防火墙不可用、磁盘过高、内存过高等。

### 5.2 流量监控要求

- 显示实时上行和下行速率，单位按 B/s、KB/s、MB/s 自动转换。
- 显示累计发送和接收流量，优先从 `/proc/net/dev` 采集。
- 支持多网卡时展示活跃网卡，默认选总流量最大的非 loopback 网卡。
- 折线图至少展示最近若干次刷新采样，不要求 V1 支持长周期存储。
- 采集失败时显示降级原因，不影响其他 Dashboard 指标。

## 6. 网站管理边界

V1 网站管理以 Nginx 为核心，不在第一阶段同时覆盖 Apache 和 Caddy。

V1 应支持：

- 探测 Nginx 是否安装、配置路径、站点 include 目录。
- 解析常见 server block，展示域名、监听端口、根目录、SSL 标记和配置文件。
- 编辑配置前创建远端备份。
- 保存后先执行 `nginx -t`，测试通过后才允许 reload。
- 测试失败时回滚或保留失败状态，禁止静默 reload。
- 反向代理配置生成可复用已有 Verdaccio proxy/Nginx 配置能力。

后续再扩展：

- SSL 证书申请和续期。
- 访问日志和错误日志分析。
- Apache/Caddy 站点管理。
- WAF 或基础防护策略。

## 7. 数据库管理边界

数据库管理需要分阶段实现，避免 V1 变成完整数据库客户端。

V1 应支持：

- 通过 systemd 和命令探测 MySQL、MariaDB、PostgreSQL、Redis 是否存在。
- 展示服务状态、版本、监听端口、数据目录线索和最近日志。
- 支持启动、停止、重启等服务级操作，并写入审计。
- 提供备份恢复入口和前置检查，但复杂备份策略可后置。

V1 不做：

- 不做完整 SQL 编辑器。
- 不默认读取或保存数据库 root 密码。
- 不做跨库迁移、主从复制配置或在线 schema 变更。
- 不做危险批量删除库表操作。

后续再扩展：

- 数据库和用户列表。
- SQL 控制台。
- 慢查询和连接数分析。
- Redis key 浏览和基础诊断。
- 定时备份、远程备份、恢复演练。

## 8. 版本路线

### V1：单服务器工作台成型

目标是把单台服务器管理做完整，优先服务真实个人开发者和小团队。

- 首页仪表盘改成宝塔类状态总览。
- 文件管理稳定可用。
- systemd 服务管理可视化。
- Nginx 网站管理基础闭环。
- 流量监控和短期趋势图。
- 项目部署和回滚基础。
- 终端、Cron、防火墙、环境变量、审计继续稳定。
- 服务器类型显示准确，腾讯云轻量应用服务器不再显示为 CVM。

### V1.5：开发者服务和数据库基础

- 数据库服务发现、状态、启停、备份恢复入口。
- GitLab CE 管理入口优化。
- Verdaccio npm 私有仓库管理优化。
- Dart/Flutter pub 配置辅助继续保留为外部 Hosted Repository 路线。
- 常见服务识别增强：mysql、mariadb、postgresql、redis、docker、gitlab、verdaccio、nginx。
- 首页服务概览和风险提示增强。

### V2：多服务器和自动化

- 多服务器总览。
- 监控告警和长周期历史。
- Docker 管理。
- 高级云资源管理：快照、云盘、计费、弹性公网 IP。
- 自动化运维任务和批量执行。
- Windows 原生版本跟进核心功能。

## 9. 安全边界

所有会修改远端状态的操作必须遵守以下约束：

- 操作前展示目标、命令预览、影响范围和恢复说明。
- 用户确认后才执行高风险写操作。
- 配置文件修改前必须创建远端备份。
- Nginx 配置修改必须先测试，再 reload。
- 数据库和防火墙默认保守，不做危险批量修改。
- 远程命令 stdout/stderr 应脱敏后再进入长期日志或报告。
- 云 API 权限不足或调用失败时降级展示，不影响 SSH 主路径。
- 服务控制、文件删除、权限修改、配置保存、部署、回滚、备份恢复都必须写入审计。

## 11. 验收标准

- 新产品定位在文档中清晰表达为“macOS 原生宝塔类服务器管理产品”。
- 文档明确区分当前已有能力、V1 要补齐和后续版本。
- 文档包含文件管理、数据库管理、网站管理、项目/服务管理、流量监控。
- 文档明确服务器类型包含腾讯云轻量应用服务器，并要求 UI 不把它显示为 CVM。
- 数据库管理和网站管理都有明确 V1 边界，避免范围过大。
- 安全边界覆盖远程写操作、配置备份、Nginx 测试、审计和云 API 降级。
