# Code of Conduct / 行为准则

[English](#english) | [中文](#中文)

## English

HHC Server Manager should be a respectful and useful open-source space. Everyone who participates is expected to help keep discussions clear, constructive, and welcoming.

### Expected behavior

- Be respectful and patient.
- Critique ideas and code, not people.
- Give actionable feedback with enough context.
- Assume good intent, especially across languages and time zones.
- Keep discussions relevant to the project.

### Unacceptable behavior

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Sharing secrets, credentials, private keys, or production server details.
- Repeatedly derailing technical discussions.

### Reporting

If you see behavior that violates this code, please contact the maintainers through a private channel or by opening a GitHub issue that does not expose sensitive information. Maintainers may remove content, close issues, or restrict participation when needed.

## 中文

HHC 服务器管理器希望成为一个友好、务实、尊重彼此的开源空间。所有参与者都应帮助保持讨论清晰、有建设性。

### 鼓励的行为

- 保持尊重和耐心。
- 讨论观点和代码，不攻击个人。
- 提供带上下文、可执行的反馈。
- 尤其在跨语言、跨时区交流中，优先假设对方是善意的。
- 让讨论保持在项目相关范围内。

### 不接受的行为

- 骚扰、恐吓、威胁或人身攻击。
- 歧视性语言或行为。
- 未经许可公开他人隐私信息。
- 分享密码、凭据、私钥或生产服务器细节。
- 反复偏离技术讨论并干扰协作。

### 报告方式

如果你看到违反行为准则的情况，请通过私密渠道联系维护者，或创建不暴露敏感信息的 GitHub issue。维护者可在必要时删除内容、关闭 issue 或限制参与。
